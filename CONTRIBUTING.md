# Contributing to BUrank

First off, thank you for considering contributing to BUrank! 🎉

BUrank is an open-source LeetCode leaderboard built for colleges to encourage competitive programming and make coding progress visible.

We welcome contributions of all sizes, from fixing typos to implementing new features.

---

# Getting Started

## 1. Fork the repository

Click the **Fork** button at the top right of GitHub.

---

## 2. Clone your fork

```bash
git clone https://github.com/YOUR_USERNAME/bennett-leetcode.git
```

---

## 3. Install dependencies

```bash
npm install
```

---

## 4. Start the development server

```bash
npm run dev
```

---

# Development Guidelines

- Keep pull requests focused on one feature or fix.
- Write clean and readable code.
- Prefer reusable components.
- Maintain TypeScript safety.
- Run the linter before submitting.

```bash
npm run lint
```

---

# Pull Request Process

1. Fork the repository.
2. Create a feature branch.

```bash
git checkout -b feature/amazing-feature
```

3. Commit your changes.

```bash
git commit -m "feat: add amazing feature"
```

4. Push your branch.

```bash
git push origin feature/amazing-feature
```

5. Open a Pull Request.

---

# Commit Convention

Please follow Conventional Commits.

Examples:

```
feat: add profile badges

fix: correct leaderboard sorting

docs: update README

refactor: simplify auth flow

style: improve navbar spacing
```

---

# Code Style

- TypeScript
- TailwindCSS
- Functional Components
- No unnecessary comments
- Small reusable components

---

# Reporting Bugs

When reporting a bug include:

- Expected behaviour
- Actual behaviour
- Steps to reproduce
- Screenshots (if applicable)

---

# Feature Requests

Please open a GitHub Issue before implementing large changes.

This helps avoid duplicate work.
